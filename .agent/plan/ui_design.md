# UI設計

## 画面構成

```
[ホーム] → [路線詳細]
```

---

## 画面1: ホーム（路線選択）

### 概要
JR東日本の路線一覧をカード形式で表示。

### 要素
- ヘッダー: アプリ名
- 路線カード一覧
  - 路線名（例: 山手線）
  - 現在の状況アイコン（🟢正常 / 🟡遅延 / 🔴運転見合わせ）
  - 簡易リスクスコア
- フッター: コンテスト表記

### アクション
- カードタップ → 画面2へ遷移

---

## 画面2: 路線詳細・検索結果（タイムライン）
`detail.html`

### 概要
検索された経路を、乗り換えや発着時刻を含めた見やすいタイムライン形式で表示。

### 要素
- **ヘッダー**: 検索区間（例: 東京 → 新宿）と出発時刻
- **サマリーカード**:
  - 出発時刻 / 到着時刻
  - 所要時間
  - 乗換回数
- **タイムライン (Timeline)**:
  - 出発駅・時刻
  - 乗車路線（ラインカラー表示）・列車種別・行き先
  - 到着駅・時刻
  - 乗り換え待機時間など
- **リスク情報（Future）**:
  - *計画*: 各タイムラインセグメントに「遅延リスク」アイコンや警告を表示する予定。

### アクション
- 戻るボタン → 検索画面へ
